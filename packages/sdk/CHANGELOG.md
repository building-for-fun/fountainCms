# Changelog

All notable changes to the FountainCMS SDK will be documented in this file.

The project follows [Semantic Versioning](https://semver.org/).

---

## [Unreleased]

### Added

- Initial SDK architecture
- Client abstraction design
- Release roadmap

---

## [0.1.0] – Initial SDK Release

### Added

- SDK client initialization with configurable base URL
- Authentication helpers (login, logout)
- Basic CRUD operations for content collections
- Centralized HTTP client handling
- TypeScript-first SDK structure

### Notes

- This release focuses on **core usability**
- API surface may change before v1.0.0
- No framework-specific bindings included

---

## [0.2.0] – Typed SDK (Planned)

### Added

- Generated TypeScript types from FountainCMS schemas
- Typed collection access methods
- Improved request/response contracts

## [0.3.0] – Framework Bindings (Planned)
